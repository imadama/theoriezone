## Feature checklist (wat vaak terugkomt = marktverwachting)

### Leerling (meest voorkomend)
- Theorie in **hoofdstukken** (vaak “kleine blokken”)
- **Quiz / deeltoets** na hoofdstuk/blok
- **Oefenexamens** (vaak “CBR-niveau / CBR-gelijk”)
- **Voortgang** zichtbaar voor leerling
- Mobielvriendelijk / multi-device
- Vaak: **NL + EN**

### Rijschool/opleider (meest voorkomend)
- Dashboard: voortgang + resultaten per leerling
- (Bij platforms) verkoopflow: codes/webshop/commissie/inkoop
- Soms: integratie met rijschoolsoftware
- Soms: white-label (logo/huisstijl)

---

## Gaps / kansen voor TheorieZone (t.o.v. wat ze publiek noemen)

1) **Exam integrity / “echte examenmodus”**
   - Jouw idee “onderbreking = beëindiging en telt als poging” is strenger. Als je dit technisch + UX goed uitwerkt (heldere regels + state machine + logging), kan dit een sterke differentiator zijn.

2) **Sterk multi-tenant beheer (governance)**
   - Concurrenten praten vooral over rijscholen/white-label. Minder zichtbaar: versiebeheer, rollout per tenant, auditing, beheer-workflows.

3) **Coach-workflow (instructeur → interventie)**
   - Veel dashboards zijn “inzicht”. Kansen: acties (aanbevelingen, triggers, taken, berichten) i.p.v. alleen rapportage.

4) **Accessibility als first-class**
   - iTheorie noemt dit expliciet (voorlezen/dyslexie/kleurenblind). Dit lijkt een hygiëne-factor te worden.

---

## Aanbevolen MVP baseline (minimaal matchen om competitief te zijn)

- Hoofdstuk-gebaseerd leren + quizzen/deeltoetsen
- Oefenexamens + examensimulatie
- Voortgangsrapportage per leerling (voor leerling + rijschool)
- NL + EN (minstens voor content UI; inhoud vertaling kan gefaseerd)
- Tenant scheiding + rollen (leerling/instructeur/tenant admin)

## Aanbevolen differentiators (kies 1–2 voor v1)

- “Strict exam mode” (anti-onderbreking) + duidelijke pogingen/regels
- White-label tenant portal (logo/huisstijl) + embed op website
- Content governance: versioning + staged rollout + audit log
